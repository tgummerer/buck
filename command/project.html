buck project

Generates the configuration files for an IDE to build the project using that IDE.

Currently, only IntelliJ is supported.

This command processes all of the project_config rules across all of the BUCK files and uses them to generate the configuration files for an IDE.

In the case of IntelliJ, the generated files include:

  • .idea/libraries/*.xml, each of which defines a library in IntelliJ. A library always corresponds to a prebuilt_jar.
  • .iml files, each of which defines a module in IntelliJ. An .iml file will be created in each directory with a BUCK file that has a project_config. A module can depend on other modules, as well as libraries.
  • .idea/modules.xml, which lists all of the IntelliJ modules in the project.

Therefore, running buck project creates all of the above files for IntelliJ in-place in the repository. This is unlike other Buck commands that generate outputs in separate directories that will be deleted by running buck clean.

Note that all files generated by buck project should be listed in .gitignore.

Parameters

  • --targets The list of targets to build a project for to generate a slice of a project containing only the code for the specified targets. This is useful for large repositories.
  • ----without-tests Indicates that Buck should build a project slice without tests (the default is to include any tests that reference the targets in the project with source_under_test).
